This example uses the ESM build with automatic vendor chunking. The browser loads chunks on demand and caches them independently, improving load times for repeat visitors.

Chunk Distribution:

vendor-react
822 KB (177 KB gzip)
vendor-codemirror
1.7 MB (579 KB gzip)
vendor-jupyter-core
1.2 MB (300 KB gzip)
vendor-mathjax
864 KB (203 KB gzip)
vendor-xterm
489 KB (119 KB gzip)
vendor-lumino
200 KB (49 KB gzip)

Benefits:

  • Independent caching: Vendor chunks rarely change and are cached separately
  • Parallel loading: Browser can load multiple chunks simultaneously
  • Faster updates: Only changed code needs to be re-downloaded

Usage

ESM Script Tag
Use type="module" to load the ESM build. The browser automatically loads required chunks. 
<!-- Configuration -->
<script type="application/json" data-jupyter-embed-config>
{
  "serverUrl": "https://oss.datalayer.run/api/jupyter-server",
  "token": "your-token-here",
  "defaultKernel": "python3"
}
</script>

<!-- ESM Build (chunks loaded automatically) -->
<script type="module" src="jupyter-embed.esm.js"></script>
